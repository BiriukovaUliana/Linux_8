##### Практична робота 8

### Завдання 8.1
Чи може виклик count = write(fd, buffer, nbytes); повернути в змінній count значення, відмінне від nbytes? Якщо так, то чому? Наведіть робочий приклад програми, яка демонструє вашу відповідь.

Так, системний виклик write() може повернути значення count, яке менше за nbytes. Це називається "частковим записом" (partial write).

Основні причини, чому це стається:
- Якщо пристрій переповнений, системний виклик запише стільки, скільки влізе, і поверне цю кількість.
- Якщо під час виконання write() надійшов сигнал, виклик може бути перерваний, записавши лише частину даних.
- Якщо файл досяг максимально дозволеного розміру для процесу.
- Якщо буфер каналу заповнений, write запише лише доступний обсяг.

lseek(fd, 3, SEEK_SET);
read(fd, &buffer, 4);
де виклик lseek переміщує покажчик на третій байт файлу. Що буде містити буфер після завершення виклику read? Наведіть робочий приклад програми, яка демонструє вашу відповідь.

Маємо послідовність байтів: 4, 5, 2, 2, 3, 3, 7, 9, 1, 5.

Позиція 0: 4

Позиція 1: 5

### Завдання 8.3
Бібліотечна функція qsort призначена для сортування даних будь-якого типу. Для її роботи необхідно підготувати функцію порівняння, яка викликається з qsort кожного разу, коли потрібно порівняти два значення. Оскільки значення можуть мати будь-який тип, у функцію порівняння передаються два вказівники типу void* на елементи, що порівнюються.
- Напишіть програму, яка досліджує, які вхідні дані є найгіршими для алгоритму швидкого сортування. Спробуйте знайти кілька масивів даних, які змушують qsort працювати якнайповільніше. Автоматизуйте процес експериментування так, щоб підбір і аналіз вхідних даних виконувалися самостійно.
- Придумайте і реалізуйте набір тестів для перевірки правильності функції qsort.

Алгоритм Quicksort у середньому працює за час O(n log n), але в найгіршому випадку його складність падає до O(n^2).
Найгірший сценарій залежить від реалізації вибору опорного елемента. У стандартній бібліотеці qsort часто використовується вибір медіани або випадкового елемента, щоб уникнути деградації на вже відсортованих масивах. 
Проте для багатьох реалізацій найгіршими є:
- Вже відсортований масив.
- Масив з однакових елементів.
- Організовані масиви, що змушують алгоритм завжди обирати найменший або найбільший елемент як pivot.

### Завдання 8.4
Виконайте наступну програму на мові програмування С:
int main() {
  int pid;
  pid = fork();
  printf("%d\n", pid);
}
Завершіть цю програму. Припускаючи, що виклик fork() був успішним, яким може бути результат виконання цієї програми?

Коли викликається fork(), створюється новий процес, який є майже повною копією батьківського процесу. Головна відмінність полягає у значенні, яке повертає ця функція:
- У батьківському процесі fork() повертає PID дочірнього процесу (це завжди число більше 0).
- У дочірньому процесі fork() повертає 0.

Оскільки після виклику fork() у системі стає два процеси, рядок printf виконається двічі (один раз у кожному процесі).

### Результат роботи програми:

Програма виведе два числа:
- 0 (від дочірнього процесу).
- Число > 0 — це PID дочірнього процесу, який отримав батько.

### Завдання 8.5 (3 варіант)
Створіть програму, яка змінює вміст відкритого файлу без переміщення вказівника позиції читання/запису.

Це завдання полягає в тому, що звичайний write() завжди переміщує покажчик позиції вперед на кількість записаних байтів. Щоб змінити вміст файлу без переміщення покажчика, у Linux існують спеціальні системні виклики — pwrite() (для запису) та pread() (для читання).

Головна відмінність pwrite від write полягає в тому, що він приймає додатковий аргумент — офсет (зсув), і при цьому не змінює стан дескриптора файлу
